Implementation: Define a Performance Metric¶

It is difficult to measure the quality of a given model without quantifying its performance over training and testing. This is typically done using some type of performance metric, whether it is through calculating some type of error, the goodness of fit, or some other useful measurement. For this project, you will be calculating the coefficient of determination, R2, to quantify your model's performance. The coefficient of determination for a model is a useful statistic in regression analysis, as it often describes how "good" that model is at making predictions.

The values for R2 range from 0 to 1, which captures the percentage of squared correlation between the predicted and actual values of the target variable. A model with an R2 of 0 is no better than a model that always predicts the mean of the target variable, whereas a model with an R2 of 1 perfectly predicts the target variable. Any value between 0 and 1 indicates what percentage of the target variable, using this model, can be explained by the features. A model can be given a negative R2 as well, which indicates that the model is arbitrarily worse than one that always predicts the mean of the target variable.

Question 3 - Training and Testing¶

What is the benefit to splitting a dataset into some ratio of training and testing subsets for a learning algorithm?
Hint: What could go wrong with not having a way to test your model?

Answer: Splitting the data into train and test helps us to test our models performance. If we do not split the data then there is a chance that we tend to overfit (or underfit) the data with our model. In predictive modeling our goal is to increase the predictive accuracy and precision on the unseen data. But if we use the same data to train and test, then we cannot evaluate our model's performance on the unseen data. For instance we can make our trained model complex enough that the training error is minimized to a maximum extent. However such over fitted models will always perform very poor on the unseen data. Hence splitting the data into train and test, and using the test data to evaluate the performance will help us to assess the model's behavior on the unseen data (the test data).

The only drawback of splitting the data is the reduction of the training data to train our models. Usually (although NOT always), as the amout of training data increases, the model's accuracy also increases.

Question 7 - Grid Search¶

What is the grid search technique and how it can be applied to optimize a learning algorithm?

Answer: Gird search is applicable to hyper parameter tuning. Hyper parameters are related to the options that can be used to build the predictive models. The hyper parameters are based on the learning algorithm. For instance in polynomial regression the maximum degree of polynomial is a hyper parameter, that can be tuned to avoid overfitting/underfitting. For boosted regression trees, we can choose different values for the tree depth and the learning rate. These 2 parameters are the hyper parameters for boosted regression trees.

While developing the machine learning models we try various combinations of the hyper parameters and choose the best combination of hyper parameters for which the cross validation error is the least on the training data. In grid search, we supply the desired values for each hyper parameter, and the grid search technique develops the algorithm using various combinations of parameters and suggests the best parameter values for which the cross validation error is minimum (the combinations are either built systematically or randomly). In sklearn, we have a package called model_selection.GridSearchCV that performs the grid search for hyper parameter tuning.

Question 8 - Cross-Validation¶

What is the k-fold cross-validation training technique? What benefit does this technique provide for grid search when optimizing a model?
Hint: Much like the reasoning behind having a testing set, what could go wrong with using grid search without a cross-validated set?

Answer: In Cross Validation (CV) method, we divide the data into k folds, use the data from one of the folds as the test data to test the model developed using the data from the remaining (k-1) folds. This process is repeated, to finally obtain k test errors. These k test errors are averaged to compute the CV error. In CV technique, we are not needed to create a test data set separately, and this means we have more data available for training, and hence we can build better models.

CV technique is used in grid search method, to choose the optimal values for the hyper parameters. Models built with different combinations of hyper parameters are compared using their CV errors and the hyper parameter combination for which the CV error is minimum is finally suggested to build the model. If CV method is not used in grid search, then we tend to tune the hyper parameters to such an extent where the overfitting of the training data could result.

Another technique of validation is to keep some training data aside and use that data to evaluate the models (developed using various combinations of hyper parameters). This type of validation is faster than the CV method. But this method will decrease the available data for training. And also this method will test the models using only a set of observations, and there might be a chance that one model might perform extremely well on the validation data, while may underperform on the unseen data in general. Such problems can be avoided by testing the models on a number of validation data sets, and averaging the errors to compare the models. The CV technique is better than the single validation set approach.

from sklearn.model_selection import GridSearchCV
from sklearn.tree import DecisionTreeRegressor
from sklearn.metrics import make_scorer

def fit_model(X, y):
    """ Performs grid search over the 'max_depth' parameter for a 
        decision tree regressor trained on the input data [X, y]. """
    
    # Create cross-validation sets from the training data
    cv_sets = ShuffleSplit(X.shape[0], n_iter = 10, test_size = 0.20, random_state = 0)

    # TODO: Create a decision tree regressor object
    regressor = DecisionTreeRegressor()

    # TODO: Create a dictionary for the parameter 'max_depth' with a range from 1 to 10
    params = {"max_depth": range(1,11)}

    # TODO: Transform 'performance_metric' into a scoring function using 'make_scorer' 
    scoring_fnc = make_scorer(score_func=performance_metric,greater_is_better=True)
    

    # TODO: Create the grid search object
    #grid = None
    grid = GridSearchCV(regressor, params,cv=cv_sets,scoring=scoring_fnc)
    #grid = GridSearchCV(regressor, params,cv=cv_sets,scoring="mean_squared_error")
    
    # Fit the grid search object to the data to compute the optimal model
    grid = grid.fit(X, y)
    
    
    #print pd.DataFrame(grid.cv_results_) 
    
    # Return the optimal model after fitting the data
    return grid.best_estimator_
